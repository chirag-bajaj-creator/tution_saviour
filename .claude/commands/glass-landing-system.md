## Step 1 - Inspect first
- Open the current landing page files and shared CSS for the target app or apps.
- Check auth entry points, CTA behavior, and any existing visual patterns that must be preserved.
- Identify what should remain shared across the three sites and what must differ.

## Step 2 - Define the page direction
- Apply the glassmorphism landing system from `.claude/skills/glassmorphism-landing-system.md`.
- Keep the design premium, minimalist, and production-ready.
- Include a strong hero visual in the first fold.
- Use product-specific sections instead of repeating the same section stack everywhere.

## Step 3 - Implement
- Update the relevant `Landing.jsx` file or files.
- Add any required landing-specific styles in the relevant `src/index.css`.
- Preserve real routes and modal login behavior.
- Keep the page responsive on mobile and desktop.

## Output standard
- Do the work, not just planning.
- Keep the page visually rich but restrained.
- Make the result feel intentional, not template-like.
